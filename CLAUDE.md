# CLAUDE.md — Конфигурация для AI-агента

## Контекст проекта
- Проект: QA Automation Framework для тестирования веб-приложения
- Язык: Java 17+
- Фреймворки: JUnit 5, Rest Assured, Selenium WebDriver
- Цель: Тестирование API + UI в едином сценарии

## Правила работы с кодом
1. **Не нарушай существующую архитектуру** без явного запроса на рефакторинг
2. **При генерации тестов**:
   - Используй Page Object Model для UI-части
   - Разделяй API- и UI-шаги логически
   - Добавляй мягкие ожидания (Explicit Waits), а не только implicit
   - Обрабатывай потенциальные исключения (timeout, element not found)
3. **При ревью кода**:
   - Ищи проблемы с thread-safety (важно для параллельного запуска)
   - Обращай внимание на hardcoded значения и пути
   - Проверяй корректность использования паттернов (Singleton, Builder и т.д.)
   - Оценивай читаемость и поддерживаемость

## Формат ответов
- Всегда структурируй ответ: проблема → объяснение → решение → пример кода
- Если не уверен — задавай уточняющие вопросы, не предполагай
- При генерации кода указывай, какие импорты нужны

## Ограничения
- Не предлагай решения, требующие внешних зависимостей без согласования
- Не меняй сигнатуры публичных методов без крайней необходимости
- При работе с тестами: каждый тест должен быть изолированным и идемпотентным

## Приоритеты при рефакторинге
1. Thread-safety и поддержка параллельного запуска
2. Читаемость и поддерживаемость
3. Производительность
4. Расширяемость
